# 小墨软件 UI Style Guide  
**Version 1.0**

本指南用于统一小墨软件在设计、开发中的 UI 视觉与交互风格，确保整体体验现代、统一、克制、有高级感。

---

# 1. 设计哲学（Design Philosophy）

## 1.1 品牌调性
小墨是一款面向学习者的智能系统，其视觉语言需体现：
- **清晰、专业、可信赖**  
- **温和与克制**（不喧宾夺主）  
- **系统性、有序性、结构感强**  
- **科技感但不炫技、未来感但不过度装饰**

## 1.2 品牌关键词
- Modern（现代）
- Minimal（极简）
- Calm（平静）
- Orderly（有序）
- Premium（高级感）
- Apple-like（苹果气质）
- Soft Light（柔光）

## 1.3 视觉抽象基调
- **光感：** 使用低对比度高光、柔和阴影、轻微渐变。  
- **留白：** 空气感是核心要素，避免密集排列。  
- **空间感：** 多使用分组、分区与卡片化结构，营造清晰层级。

# 2. 颜色规范（Colors）

## 2.1 主色（Brand Color）
- **墨蓝（Primary）**：`#2A4CFF`（或接近苹果蓝）  
- 用途：主要按钮、关键交互点、选中状态  

## 2.2 辅助色（Secondary）
- **柔紫（Secondary）**：`#6E6BEF`  
- 用途：强调但不过度吸引注意，适用于 badge、提示标签  

## 2.3 中性色（Neutrals）
- 文本主色：`#1A1A1A`  
- 文本副色：`#5E5E5E`  
- 分割线：`#E5E5E5`  
- 背景色：`#F7F8FA`（柔灰接近苹果的 “Canvas”）  
- 卡片背景：`#FFFFFF`  

## 2.4 高级感原则
- 避免高饱和色  
- 只在绝对必要的地方使用亮色（如 CTA）  
- 保持“安静的配色”，减少心理负担  

## 2.5 禁忌
- 禁用纯黑纯白（以免刺眼）  
- 禁用强烈渐变  
- 禁用多色混搭  
- 禁用粗糙阴影  

---

# 3. 字体与排版（Typography）

## 3.1 字体
- **中文：** SF Pro SC（或苹方 / 思源黑体作为 fallback）  
- **英文：** SF Pro Text / SF Pro Display  

## 3.2 层级规范
| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | 28–32px | 600 | 页面标题 |
| H2 | 20–24px | 600 | 大模块标题 |
| H3 | 16–18px | 500 | 小标题、分组标题 |
| Body | 14–16px | 400 | 正文内容 |
| Caption | 12–13px | 400 | 辅助说明 |

## 3.3 行距
- 主体内容：1.45–1.6  
- 多段文字保持宽松、舒适的阅读节奏  

## 3.4 原则
- 排版保持“轻”与“静”  
- 行距不宜紧凑  
- 避免过多的粗体

# 4. 组件风格（Components）

## 4.1 Buttons
- 形状：圆角 8–10px  
- 主按钮：实心主色 + 柔光外描边  
- 次按钮：浅灰描边 + 白底  
- Hover：轻微提升亮度（不增加饱和度）  
- Active：增加轻微内阴影  

## 4.2 Cards
- 完全白底  
- 圆角 12px  
- 阴影：**极浅柔光**（0px 4px 12px rgba(0,0,0,0.06)）  
- 内部保持较大留白（24–32px padding）

## 4.3 Tabs
- 使用浅底 + 宽松间距  
- 选中态采用半透明高光 + 精细描边  
- Hover：轻微透明度变化（不跳色）

## 4.4 Inputs
- 圆角 10px  
- 边框：`#DDDDDD`  
- Focus：苹果风蓝色外描边 + 柔和 glow  
- 行距与文本高度保持舒适阅读

---

# 5. 布局与空间（Layout & Spacing）

## 5.1 栅格系统
- 主屏幕：12 列 Grid  
- 侧栏布局：左侧固定 240px，右侧自适应  

## 5.2 间距规范
| 类型 | 数值 |
|------|------|
| 小间距 | 8px |
| 中间距 | 16px |
| 大间距 | 24px |
| 版心边距 | 40px+ |

## 5.3 原则
- 单屏内容层级不超过 3 层  
- 以「留白空间」区分模块，而非粗线条  
- 保持视觉呼吸感，不拥挤  

---

# 6. 动效（Motion）

## 6.1 风格原则
- 自然（Natural）  
- 柔和（Soft）  
- 不抢视线（Subtle）  
- 可解释当前状态（Meaningful）  

## 6.2 动效规范
- 页面切换：200–260ms 淡入淡出  
- 折叠展开：180–220ms 高度过渡  
- 悬停：即时响应（50–80ms）  
- 模态开合：向上轻推 + 轻微弹性  

---

# 7. 图标与插图（Icons & Illustrations）

## 7.1 图标
- 极简线性图标  
- 2px 线宽  
- 圆角拐点  
- 减少填充图标的使用（保持轻量）  

## 7.2 插图风格
- 柔光、半透明、轻渐变  
- Apple-like 玻璃感、模糊效果可选  
- 色彩轻且干净  
- 避免插图喧宾夺主  

## 7.3 图标统一性
- 所有图标必须来自同一套库（或定制）  
- 禁止混搭不同线条风格  

---

# 8. 常见页面模板（Page Templates）

## 8.1 导航栏（Sidebar）
- 左侧固定  
- 背景白或浅灰  
- 图标 + 标签排列  
- hover 使用浅色铺底，不改变文字颜色  
- 分割线用浅灰（不可太深）  

## 8.2 学习页面（Reader + Notes）
- 左：阅读器（大留白）  
- 右：笔记区（可折叠）  
- 阅读器高亮与选框采用苹果蓝  
- 文档内元素需遵守同一排版体系  

## 8.3 知识库（Knowledge Base）
- 分区卡片化  
- 使用标签系统区分内容类型  
- 空状态采用柔和插图与简短提示文案  
- 强调“整洁、有序、专业感”  

## 8.4 任务树（Task Tree）
- 节点使用轻卡片  
- 圆角 10–12px  
- 节点之间的链接线细而柔和  
- 任务 hover 时整体略微提升阴影（Apple 风）  
- 保持可视化层级清晰，并以留白分层  

---

# 结语
本 UI Style Guide 是小墨软件的统一视觉语言标准。  
所有 UI、插图、动效、前端组件均应严格遵循本规范，以确保产品的专业感、统一性和长期扩展性。

如需进一步扩展组件、动效或更细的设计标准，请在此基础上补充。
